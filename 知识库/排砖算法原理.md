# 排砖算法原理

## 核心概念

### 排砖问题本质
在给定的封闭区域内，以特定规格的矩形砖块 + 灰缝，按某种排列图案尽可能填满区域。

### 输入参数
- 区域边界（闭合多段线）
- 砖规格：长(L) × 宽(W) × 厚(T)
- 灰缝：水平(Jh) × 垂直(Jv)
- 铺贴方式：直铺/工字/人字/斜铺
- 起铺点：居中/角落/手动
- 避让区域：门窗洞口

### 输出结果
- 每块砖的位置(x,y)、旋转角(θ)、实际尺寸
- 切割砖标记 + 废料统计
- 材料用量：整砖数、切割砖数、损耗率

## 四种铺贴方式算法

### 1. 直铺（Straight）
```
for r in 行:
  for c in 列:
    x = startX + c × (L + Jv)
    y = startY + r × (W + Jh)
    // 边界裁剪 + 避让检测
```

### 2. 工字铺（Brick Bond）
```
for r in 行:
  offset = (r % 2) × L × ratio  // 奇数行错开
  for c in 列:
    x = startX + offset + c × (L + Jv)
    y = startY + r × (W + Jh)
```

### 3. 人字铺（Herringbone）
```
砖块以 ±45° 交替排列
对角线方向为主排列方向
step = √(L²+W²) + Jh
```

### 4. 斜铺（Diagonal）
```
整体图案旋转 45°
使用旋转矩阵变换坐标
```

## 切割优化策略

### 最小尺寸约束
- 规则：切割后砖块 ≥ 原砖 × minRatio（如 1/3）
- 不满足 → 优先调整起铺点（重新计算）
- 仍不满足 → 标记为不合格切割，根据模式丢弃或保留

### 优先整砖
- 计算不同起铺点的整砖数量
- 选择整砖最多的方案
- 在中心附近微调（±step/2 范围内搜索最优）

## 区域分析（射线法）

判断点是否在多边形内：
```
从点向右发射水平射线
与多边形边界的交点数为奇数 → 在内部
```
